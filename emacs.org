* straight.el Setup

[[https://github.com/radian-software/straight.el][straight.el]] is a next-generation package manager for Emacs that replaces the built-in package.el. Instead of downloading prebuilt tarballs, it installs packages by cloning their Git repositories directly, giving you precise control over versions. This makes your Emacs configuration fully reproducible — you can lock packages to exact commits and share a consistent setup across machines. It also makes local package development easy, since you're just editing the live Git checkout. It integrates well with use-package as a drop-in alternative to :ensure.

 The following code snippet downloads and installs straight.el. In case this does not work one can clone the straight.el repo from github into ~/.emacs.d/straight/repos.

#+begin_src emacs-lisp

  (defvar bootstrap-version)
  (let ((bootstrap-file
         (expand-file-name
  	"straight/repos/straight.el/bootstrap.el"
  	(or (bound-and-true-p straight-base-dir)
  	    user-emacs-directory)))
        (bootstrap-version 7))
    (unless (file-exists-p bootstrap-file)
      (with-current-buffer
  	(url-retrieve-synchronously
  	 "https://radian-software.github.io/straight.el/install.el"
  	 'silent 'inhibit-cookies)
        (goto-char (point-max))
        (eval-print-last-sexp)))
    (load bootstrap-file nil 'nomessage))

  (straight-use-package 'use-package)

#+end_src

  		      :font pl/default-font
  		      :height pl/default-font-size))

  (use-package mixed-pitch

  			:family pl/variable-pitch-font
  			:height pl/variable-pitch-font-size)))

  		      :height pl/fixed-pitch-font-size))

** Setup transparency

Working in a slightly transparent frame is neet! So let's configure it. Unfortunately, it does work a little different on windows, since the variable alpha-background is not supported there. 

#+begin_src emacs-lisp

  (defun pl/set-transparency ()
  "Set the transparency according to the platform."
  (cond
   ((eq system-type 'windows-nt)
    (set-frame-parameter (selected-frame) 'alpha '(95 . 100))
    (add-to-list 'default-frame-alist '(alpha . (95 . 100)))
    (message "Running on Windows"))
   ((eq system-type 'gnu/linux)
    (set-frame-parameter nil 'alpha-background 95)
    (add-to-list 'default-frame-alist '(alpha-background . 95))
    (message "Running on GNU/Linux"))
   (t
    (message "Unknown operating system"))))

#+end_src

** all-the-icons

[[https://github.com/domtronn/all-the-icons.el][all-the-icons]] is an Emacs library that provides icon rendering using several icon fonts (FontAwesome, Material Design Icons, file-icons, etc.), allowing other packages to display icons in the modeline, file explorers, and elsewhere. It was the dominant icon library for Emacs before being largely superseded by nerd-icons.

#+begin_src emacs-lisp

  (use-package all-the-icons
    :straight t
    :if (display-graphic-p))

#+end_src


** nerd-icons

[[https://github.com/rainstormstudio/nerd-icons.el][nerd-icons]] is an Emacs library that renders icons from Nerd Fonts within Emacs buffers and UI elements. It is used as a dependency by other packages (like doom-modeline and dirvish) to display file type icons, mode indicators, and other glyphs. It requires a Nerd Font to be installed on your system to display correctly.

#+begin_src emacs-lisp

  (use-package nerd-icons
    :straight t)

#+end_src


** doom-modeline

[[https://github.com/seagle0128/doom-modeline][doom-modeline]] is a fancy, minimal modeline (the status bar at the bottom of Emacs windows) inspired by Doom Emacs. It provides a sleek, icon-rich modeline that displays useful information such as:

- Current buffer name and modification status
- Git branch and diff statistics (via magit/git-gutter)
- Major mode with an optional icon
- Line/column number and buffer position
- LSP/flycheck/flymake error and warning counts
- Active virtual environment (Python, etc.)
- Battery, time, and other optional segments

It relies on nerd-icons (or previously all-the-icons) for the icons, and is designed to be fast and lightweight compared to heavier alternatives like Spaceline. Like doom-themes, it originated in Doom Emacs but works as a standalone package in any Emacs configuration.

#+begin_src emacs-lisp

  (use-package doom-modeline
    :straight t
    :after (all-the-icons nerd-icons)
    :init (doom-modeline-mode 1))

#+end_src


** doom-themes

[[https://github.com/doomemacs/themes][doom-themes]] is a collection of Emacs themes inspired by Doom Emacs. It provides a set of carefully crafted color themes (like doom-one, doom-dracula, doom-molokai, etc.) with consistent styling across many major and minor modes. It also includes optional extras like org-mode fontification tweaks and neotree/treemacs icon support. The package originated from the Doom Emacs configuration framework but works as a standalone package in any Emacs setup.

** Rainbow delimiters

[[https://github.com/Fanael/rainbow-delimiters][rainbow-delimiters]] is an Emacs minor mode that colorizes nested parentheses, brackets, and braces by depth — each nesting level gets a distinct color, making it much easier to visually match pairs and understand code structure. It's especially popular in Lisp-family languages.

#+begin_src emacs-lisp

** Paredit

[[https://www.emacswiki.org/emacs/ParEdit][paredit]] is a minor mode for structured editing of Lisp code. Its core idea is that parentheses (and other delimiters) are always kept balanced — you can never accidentally create mismatched pairs. It provides commands for navigating, slurping (pulling in neighboring expressions), barfing (pushing out expressions), splitting, joining, and raising S-expressions, making it much easier to reshape Lisp code structurally rather than editing raw characters.

#+begin_src emacs-lisp
  (use-package paredit
    :straight t)

  (autoload 'enable-paredit-mode "paredit" "Turn on pseudo-structural editing of Lisp code." t)
  (add-hook 'emacs-lisp-mode-hook       #'enable-paredit-mode)
  (add-hook 'eval-expression-minibuffer-setup-hook #'enable-paredit-mode)
  (add-hook 'ielm-mode-hook             #'enable-paredit-mode)
  (add-hook 'lisp-mode-hook             #'enable-paredit-mode)
  (add-hook 'lisp-interaction-mode-hook #'enable-paredit-mode)
  (add-hook 'scheme-mode-hook           #'enable-paredit-mode)
  ;; (add-hook 'c-mode-hook                #'enable-paredit-mode)
  (add-hook 'rustic-mode-hook           #'enable-paredit-mode)

#+end_src



** which-key

[[https://github.com/justbur/emacs-which-key][which-key]] is a minor mode that displays a popup showing available keybindings whenever you pause after entering an incomplete key sequence. For example, if you press C-x and wait a moment, a panel appears listing all the commands accessible from that prefix. It's a great discoverability tool, especially for users still learning Emacs keybindings. As of Emacs 30.1, it is included with Emacs itself and no longer needs to be installed separately.

** Savehist

[[https://www.gnu.org/software/emacs/manual/html_node/emacs/Saving-Emacs-Sessions.html][savehist]] is a built-in Emacs minor mode that persists your minibuffer history across sessions. It periodically saves inputs you've typed at minibuffer prompts — such as commands, search strings, file paths, and other queries — to a file on disk. When you restart Emacs, that history is restored, so you can navigate back to previous inputs with M-p / M-n without having to retype them.

#+begin_src emacs-lisp

  (use-package savehist
    :straight t
    :init
    (savehist-mode))

#+end_src


** Marginalia

[[https://github.com/minad/marginalia][Marginalia]] adds rich, colorful annotations to Emacs minibuffer completion candidates — for example, showing docstrings next to commands, file permissions next to files, or symbol types (function, variable, command) next to Elisp symbols. It automatically detects the category of what you're completing and picks an appropriate annotator for it.

    :init

    ;; Marginalia must be activated in the :init section of use-package such
    ;; that the mode gets enabled right away. Note that this forces loading
    ;; the package.
    (marginalia-mode))

#+end_src

** Emacs

The package /emacs/ is not a real installable package — it's a pseudo-package used as a convenient catch-all in use-package declarations. It lets you apply use-package's structured keywords (:init, :config, :bind, :hook, etc.) to general built-in Emacs settings that don't belong to any specific package. It also supports a :ensure value to declare a minimum required Emacs version. Essentially, it's just a tidy way to organize global Emacs configuration within the same use-package style as the rest of your init file.

#+begin_src emacs-lisp
  ;; A few more useful configurations...
  (use-package emacs
    :custom
    ;; Enable context menu. `vertico-multiform-mode´ adds a menu in the minibuf
    ;; to switch display modes.
    (context-menu-mode t)
    ;; Support opening new minibuffers from inside existing minibuffers.
    (enable-recursive-minibuffers t)
    ;; Hide command in M-x which do not work in the current mode.
    ;; Vertico commands are hidden in normal buffers. This setting is useful
    ;; beyond Vertico
    (read-extended-command-predicate #'command-completion-default-include-p)
    ;; Do not allow the cursor in the minibuffer prompt
    (minibuffer-prompt-properties
     '(read-only t cursor-intangible t face minibuffer-prompt))
    
    ;; TAB cycle if there are only few candidates
    ;; (completion-cycle-threshold 3)
    
    ;; Enable indentation+completion using the TAB key.
    ;; `completion-at-point' is often bound to M-TAB.
    (tab-always-indent 'complete)
    
    ;; Emacs 30 and newer: Disable Ispell completion function.
    ;; Try `cape-dict' as an alternative.
    (text-mode-ispell-word-completion nil))

#+end_src


** Consult

[[https://github.com/minad/consult][Consult]] provides a rich set of search and navigation commands built on top of Emacs' standard completing-read. ** inheritenv

The [[https://github.com/purcell/inheritenv][inheritenv]] Emacs package (by Steve Purcell) fixes a common annoyance where Emacs commands that spawn subprocesses in temporary buffers ignore the buffer-local environment of the buffer you invoked them from. Normally, things like shell-command-to-string always use Emacs-wide defaults for process-environment and exec-path, which means tools set up per-project (e.g. via envrc, direnv, or with-editor) can be missed — leading to "executable not found" errors or the wrong version of a binary being used.

The package provides two main tools: an inheritenv macro that library authors can wrap around code running processes in temp buffers so those processes inherit the caller's buffer-local env, and an inheritenv-add-advice macro that end users can apply to existing commands (like shell-command-to-string) to retrofit the same behavior.

#+begin_src emacs-lisp

     ;; install required inheritenv dependency:
   (use-package inheritenv
     :straight (:type git :host github :repo "purcell/inheritenv"))

#+end_src


** eat

The [[https://codeberg.org/akib/emacs-eat][eat]] Emacs package (short for "Emulate A Terminal," by Akib Azmain Turja) is a terminal emulator written entirely in Emacs Lisp. Despite being pure Elisp, it's notably fast — reportedly more than three times faster than the built-in term — and can run most full-screen terminal programs, including Emacs itself.
Its main selling points are complete mouse support, clipboard integration between Emacs and the terminal, reduced flickering, and tight integration with Eshell (so you can run full-screen TUI programs like htop or vim directly inside Eshell). It's available on NonGNU ELPA via M-x package-install RET eat RET.

** vterm

The [[https://github.com/akermu/emacs-libvterm][vterm]] Emacs package (emacs-libvterm, by akermu) is a fully-featured terminal emulator for Emacs built on top of libvterm, a C library. Because the heavy lifting happens in compiled code rather than Elisp, it's dramatically faster than built-ins like term, ansi-term, or shell — rendering text up to two orders of magnitude faster and handling large outputs smoothly.

It supports essentially all of xterm's escape codes, so it runs ncurses programs like vim, htop, tmux, etc. without issues — basically feeling like a standalone terminal inside an Emacs buffer. It can also track the current directory and shell prompt (with some shell-side setup), and supports a message-passing mechanism that lets shell commands invoke Emacs functions (e.g. open files in the running Emacs). The main trade-off is that it requires a C module to be compiled, and it's not as deeply Emacs-native as Eshell, so some of Emacs's usual editing keybindings don't apply inside the buffer.

#+begin_src emacs-lisp
   ;; for vterm terminal backend:
   (use-package vterm :straight t)

#+end_src

* The jinx spell checker

[[][Jinx]] is Daniel Mendler's fast, just-in-time spell checker that uses Enchant, and it already knows how to restrict itself to comments and docstrings inside prog-mode buffers — so you mostly just need to turn it on in the right places.

A few notes on how this behaves for your stack:

Org-mode gets full prose checking (headings, paragraphs, list items). Jinx automatically skips source blocks, #+ keywords, links, and verbatim/code spans, so it won't squawk at your #+BEGIN_SRC blocks.
For python-mode, rust-mode / rust-ts-mode, c-mode / c-ts-mode, and c++-mode / c++-ts-mode, the prog-mode hook is enough — Jinx walks the buffer's font-lock faces and only checks text fontified as font-lock-comment-face or font-lock-doc-face. If you'd rather not spell-check regular string literals (e.g., in Python), drop font-lock-string-face from jinx-include-faces.

If you want an even simpler setup, replace the three hooks with (global-jinx-mode 1) in :config — Jinx is careful about which modes it activates in.

One prerequisite: Jinx links against the Enchant C library, so on first install Emacs will compile its module. You'll want libenchant and at least one backend dictionary (e.g., Hunspell or Aspell with en_US) available on your system. On Debian/Ubuntu: sudo apt install libenchant-2-2 libenchant-2-dev hunspell hunspell-en-us. On macOS with Homebrew: brew install enchant.
To add another language later, M-x jinx-languages lets you toggle dictionaries per-buffer, and saving a word via jinx-correct (M-$) persists it to your personal dictionary.

#+begin_src emacs-lisp

  ;; --- Jinx ---
  (use-package jinx
    :straight t
    :hook ((org-mode  . jinx-mode)
           (text-mode . jinx-mode)
           (prog-mode . jinx-mode))   ; comments/strings only in code
    :bind (("M-$"   . jinx-correct)
           ("C-M-$" . jinx-languages))
    :custom
    (jinx-languages "en_US")
    :config
    (setq jinx-include-faces
          '((prog-mode font-lock-comment-face
                       font-lock-doc-face
                       font-lock-string-face)
            (conf-mode font-lock-comment-face)))

    (setq jinx-exclude-regexps
          '((t "[A-Z]+\\>"
               "\\<[[:upper:]][[:lower:]]+[[:upper:]]\\w*\\>"
               "\\w*[0-9]\\w*"
               "[a-z]+_[a-z_]+"
               "https?://\\S-+"))))

#+end_src
